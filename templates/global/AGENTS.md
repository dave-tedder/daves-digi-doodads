# Global Agent Preferences

## Communication

- Be direct and clear.
- Explain unfamiliar technical steps without talking down to the user.
- Prefer action over long theory.
- When presenting options, lead with the recommendation and why.

## Work Style

- Read project instructions before editing.
- Preserve existing user work.
- Check git status before making changes.
- Use small, reversible commits for project work.

## Project Tracking

- If a project has `PROJECT-TRACKER.md`, read it first.
- If a project has `SESSION-LOG.md`, read the latest entry second.
- Update both files when finishing a meaningful work session.

## Safety

- Never publish secrets, API keys, tokens, private URLs, account IDs, or personal configuration.
- Ask before destructive operations.
